Claude Artifacts: The Complete Guide

AI · Tools · Build, share, embed

The Big Idea

Artifacts are interactive outputs Claude renders in a split-pane preview beside your conversation. Instead of showing raw code, Claude shows the live, working result — a calculator you can click, a chart with real data, a diagram, a small game. Ask in plain language, watch it run, then share or embed it.

💡 Key insight

Artifacts, Projects, and Claude Code solve different jobs: Projects store persistent context (files, instructions); Claude Code edits local codebases from the CLI; Artifacts are shareable browser previews generated from a conversation.

How They're Triggered

Artifacts appear automatically when Claude decides the output benefits from rendering. You can also ask explicitly: "Build this as an artifact" or "Show it in the preview pane."

Embedding

Use Claude's native embed flow with an allowed-domains configuration, or drop the artifact into an iframe:

html 
<iframe src="URL" width="100%" height="600"
  frameborder="0" sandbox="allow-scripts allow-same-origin
  allow-forms" loading="lazy"></iframe>

Limitations to Know

⚠️ Watch out
  • localStorage fails — sandboxed iframes can't access browser storage; use in-memory variables or a backend.
  • Single file only — no multi-file projects; everything must be inlined.
  • CDN allowlist — corporate firewalls may block external resources.
  • AI-powered artifacts require login — API-calling artifacts can't be fully public.
  • Mobile quirks — responsive design needs explicit testing.

Artifacts are where Claude stops describing and starts showing. Ask for the working thing, watch it render, and — with the 2026 live-data update — wire it to your real data and hand it to someone with a link.
