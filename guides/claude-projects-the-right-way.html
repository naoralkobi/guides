How to Use Claude Projects the Right Way

AI · Productivity · A workspace setup

The Big Idea

Projects turn Claude from a scattered chat tool into an organized workspace. Instead of hunting through endless conversation history, you give one project a name, a set of standing instructions, and a small curated knowledge base — and every chat inside it inherits that context automatically.

💡 Key insight

Chats inside a project don't share information with each other. Only the instructions and the knowledge base carry across. If an insight needs to persist between chats, put it in the knowledge base — not in a message.

The Setup

1Create a project with a real name

Go to claude.ai/projects → New Project. Give it a descriptive name like "Q3 2026 Product Launch — Email Sequence", not "Emails". Add an optional description.

💡 Insight

The name and description are for you, not for Claude — they're how you find the right workspace later. On Team/Enterprise plans you also set visibility (private or organization-wide) here.

2Write the custom instructions

Open Instructions inside the project. These apply to every chat in the workspace automatically, so write them as a detailed brief, not vague guidelines. Cover four areas:

AreaWhat to specify
Role / audienceWho consumes the output (a client, your team, beginners).
ToneFormal, conversational, technical, or plain-language.
FormatLength, headings, code blocks, structure preferences.
RulesCitation standards, assumptions to make, language conventions.

Example: for a blog-drafting project — "Default to 1,200 words, use H2/H3 headings only, avoid em dashes, end each section with a one-sentence takeaway." Now you never repeat that in a new chat.

3Build the knowledge base

Under Files, click + and upload PDFs, Word docs, spreadsheets, code, or plain text. The content becomes available across all chats in the project.

💡 Paid-plan advantage

On paid plans, RAG (Retrieval Augmented Generation) engages automatically as the knowledge base approaches the context limit, expanding effective capacity up to ~10x. Free users can create up to five projects but don't get RAG scaling.

Curate it like a reference library:

  • Upload only current, relevant documents.
  • Replace outdated versions instead of stacking duplicates.
  • Use clear, descriptive filenames.
  • Five strong docs beat forty scattered ones.

4Organize the chats

Each chat is independent, so treat chats as tasks:

  • Create a separate chat for each distinct task.
  • Rename auto-generated titles for clarity.
  • Move stray chats from your history into the project via the dropdown menu.
  • Pin frequently-used projects to the left sidebar.
  • Archive completed projects to cut clutter.

5Share it (Team / Enterprise)

Click Share project next to the project name, then add emails individually or paste a list. Two permission levels:

LevelWhat they can do
Can useView contents and work within the project; no editing.
Can editModify instructions, knowledge, and manage members.

Shared projects appear under the Shared with me tab.

⚠️ Watch out

Archiving a shared project resets all permissions to private. You can still open archived chat history, but you must unarchive a project before deleting it.

Working Habits That Pay Off

  • One project per task — don't combine unrelated work.
  • When Claude repeats a mistake, update the project instructions instead of correcting it every time.
  • Good fits: client work, thesis chapters, a job search, book drafts, recurring reports.
  • If multi-chat work hinges on one insight, add that insight to the knowledge base.

Same Claude, completely different experience. A project isn't a folder of chats — it's a standing brief plus a reference library, so every conversation starts already knowing the job.
