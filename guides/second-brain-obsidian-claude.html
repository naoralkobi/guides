Building a "Second Brain" with Obsidian + Claude

AI · Productivity · A ten-step setup

The Big Idea

ToolRoleWhy it matters
Obsidian The storage — a free notes app keeping plain-text files on your own computer. Notes link to each other → they form a visible graph of your ideas. The files are yours forever, with no cloud lock-in.
Claude The brain — reads the whole vault. Files new information where it belongs, links it to what's already there, and answers questions across everything you've saved.
💡 Key insight

It's just text files. You're not locked to any one AI. Point a different model at the vault next year and it still works — you own the brain, not the tool.

The 10 Steps

1Install Claude Desktop

Download from claude.com/download → install → sign in. This needs a paid plan (Pro, $20/mo; the free tier won't work). Click the Code tab — that's Claude Code, the version that can read and write files on your machine.

2Install Obsidian + create your vault

Download from obsidian.md (free) → Create new vault → name it brain. That folder is your second brain.

Try the core mechanic: type [[goals]] — the bracketed word becomes a link. Notes pointing at notes. Claude does this linking automatically later.

3Open the connection (Obsidian plugin)

Settings → Community plugins → turn on → Browse → search Local REST API → Install → Enable. Open its settings and copy the API Key.

Keep Obsidian running — the connection only works while it's open.

4Connect Claude to the vault

In the Claude Code tab, paste the following (swap in your key):

bash 
claude mcp add-json obsidian-vault '{ "type": "stdio", "command": "uvx", "args": ["mcp-obsidian"], "env": { "OBSIDIAN_API_KEY": "PASTE-YOUR-KEY-HERE", "OBSIDIAN_HOST": "127.0.0.1", "OBSIDIAN_PORT": "27124" } }'
⚠️ Watch out

Don't include the word "Bearer" — paste only the key string.

Test it: "list every file in my Obsidian vault." If Claude reads your notes back, it works.

5Load yourself into the brain

Use case: stop re-explaining who you are every session.

Make Claude interview you. Paste:

prompt 
You are setting up my second brain. Interview me ONE question at a
time to build my profile. Ask about: who I am and what I do, my goals
for this year, how I want you to talk to me, my strengths and
weaknesses, and my current projects. Wait for each answer before the
next question. When finished, write everything into a file called
CLAUDE.md at the vault root, structured with headers, so you load it
automatically every session.

Saved to CLAUDE.md, loaded every session.

6Create a project with structure

Use case: keep Claude focused on one job.

Paste:

prompt 
Create a project folder in my vault called youtube-channel. Inside it,
create four folders: Inputs, Process, Outputs, Feedback. Then write a
CLAUDE.md inside that project folder describing what this project is,
its one goal, and your specific role in helping me hit it.

The pipeline: ideas → Inputs · Claude works in Process · finished work → Outputs · results/metrics → Feedback. Repeat per area.

7Open a single project to work in it

💡 Insight

The big vault plans. A single project ships.

In Obsidian: vault name (bottom-left) → Manage vaults → Open folder as a vault → pick the project folder → Trust. Claude reads only that project's CLAUDE.md.

8Build a skill (never repeat yourself)

Use case: anything you do more than once → a saved workflow.

Paste:

prompt 
I want to turn this into a reusable skill. Here is how I do [the task],
with an example: [paste your example or steps]. Save this as a markdown
skill file inside this project's skills folder, with a clear name and a
description of when to trigger it.

Next time, say "run the [name] skill." Build one per repeated task.

9Wire in live data

Use case: connect what changes in real time.

bash 
claude mcp add google-workspace uvx workspace-mcp --tools calendar

Then:

prompt 
Read my calendar for today, log what I committed to in each meeting
into my tasks project, and flag anything without a clear next step.

Add Gmail, Slack, and Notion the same way. Always grant read-only where you can.

10Put it on autopilot

Use case: a brain that maintains itself overnight.

Claude Desktop → Schedule tab → + New task:

FrequencyDaily, 7:00am
Folderyour vault
PromptCheck my vault. File anything new sitting in Inputs folders into the right place and link it. Flag notes that have gone stale. Write me a 3-line summary of what changed overnight.

The One Rule You Never Break

⚠️ Keys, not prompts

Telling an agent "don't delete this" is a suggestion, not a safety setting. If it technically can delete a file or send an email, assume one day it will.

Control access at the permission level — read-only, scoped keys — not with words.
