Building Claude Skills: A Complete How-To

AI · Claude · Build your own skill

The Big Idea

A Claude Skill is just a folder of instructions that loads automatically across Claude.ai, Claude Code, and the API. No models, no paid add-ons — open-source markdown plus optional supporting files. The trick that makes them efficient is progressive disclosure: only what's needed is loaded at each moment.

💡 The three disclosure layers
  • YAML frontmatter — always loaded (~100 tokens). The name + description that tell Claude when to reach for the skill.
  • SKILL.md body — loaded when the skill is judged relevant.
  • references/ and assets/ — loaded only when the body instructs Claude to open them.

The File Structure

text 
your-skill-name/
├── SKILL.md            # Required - case-sensitive
├── scripts/            # Optional - executable code
│   └── process_data.py
├── references/         # Optional - documentation, loaded on demand
│   └── style-guide.md
└── assets/             # Optional - templates
    └── template.md

The Naming Rules That Trip People Up

⚠️ Get these exact
  • Folder name: kebab-case only, and it must match the name field in frontmatter exactly.
  • SKILL.md: case-sensitive — skill.md or SKILL.MD won't be recognized.
  • No README.md inside the skill folder (the human-readable README goes at the repo root).
  • Reserved words: the name can't contain "claude" or "anthropic".
  • No XML brackets in frontmatter (security restriction).

The Frontmatter

Minimal required form:

yaml 
---
name: your-skill-name
description: What it does. Use when [specific trigger phrases].
---

Full form with optional fields:

yaml 
---
name: your-skill-name
description: [Action] + [When to use] + [Key capabilities] (max 1024 chars, no XML)
license: MIT
compatibility: Environment requirements (max 500 chars)
metadata:
  author: Your Name
  version: 1.0.0
  mcp-server: optional-server-name
---
💡 The description is everything

It's the only part always in context, so it decides whether the skill ever triggers. Be specific about tasks and trigger phrases.
Bad: "Helps with design files."
Good: "Analyzes Figma files and generates developer handoff docs. Use when the user uploads .fig files or asks for 'design specs'."

The Body of SKILL.md

Structure it as instructions, examples, and troubleshooting. Three writing practices make it reliable:

  1. Exact commands with their expected output — not vague directives.
  2. Error handling for foreseeable failures.
  3. Clear file references using exact paths.

Worked Example: a Blog-Writing Skill

Use case: writers want consistent house style without re-pasting the guidelines every time.

structure 
blog-content-writer/
├── SKILL.md
├── references/
│   └── style-guide.md
└── assets/
    └── post-template.md
SKILL.md frontmatter 
---
name: blog-content-writer
description: Drafts blog posts following the company style guide.
  Use when the user asks to "write a blog post", "draft content for the blog",
  "create a post", or produce long-form content for publication.
license: MIT
metadata:
  author: Content Team
  version: 1.1.0
---

The body's workflow: (1) load references/style-guide.md before drafting, (2) clarify the brief — topic, audience, word count, goal, (3) draft applying the guidelines, (4) run a 7-item quality checklist, (5) return the draft with a summary block. The style guide only loads when Step 1 tells Claude to read it — progressive disclosure in action.

Installing It

Claude Code — global 
mkdir -p ~/.claude/skills
cp -r blog-content-writer/ ~/.claude/skills/
ls ~/.claude/skills/
Claude Code — project-level 
mkdir -p ./.claude/skills
cp -r blog-content-writer/ ./.claude/skills/
ls ./.claude/skills/

For Claude.ai: zip the skill folder, then go to Settings → Capabilities → Skills → upload the zip.

Testing the Trigger

The hardest part of a skill isn't the instructions — it's making it fire at the right time and stay quiet otherwise. Build a test matrix:

Should trigger onShould NOT trigger on
"Write a blog post about…""Summarize this article"
"Draft content for…""Help me fix this code"
"Create a post following our style…""Write an email"

Aim for 90%+ automatic triggering on relevant requests. After any edit to SKILL.md or the frontmatter, rerun the full matrix before distributing.

Common Problems

ProblemCauseFix
Never triggersVague description, missing phrasesAdd specific trigger language
Triggers constantlyDescription too broadAdd explicit "Do NOT use when" conditions
Instructions ignoredVague directivesRequire exact commands and outputs
Works in Claude.ai, fails in CodeMissing dependenciesDocument in the compatibility field
Inconsistent outputToo flexibleAdd a quality checklist with self-verification
💡 Key insight

The official guide promises a working skill in 15–30 minutes: start with one concrete use case, define trigger conditions before writing instructions, test triggering first, then iterate on the hardest task until it works — and extract that approach into the skill.

A skill is the cure for re-teaching Claude the same workflow. Write it once, name its trigger precisely, test that it fires, and it becomes a capability you carry into every conversation.
