Getting Started with the Claude Agent SDK

AI · Agents · Build a CLI agent

The Big Idea

The Claude Agent SDK, built on Claude Code, lets you create, extend, and customize applications powered by Claude. It connects to your local environment, gives Claude access to tools, and orchestrates complex workflows — coding, research, note-taking, automation. This guide builds TrendSmith, a market-research CLI that combines Claude Sonnet 4.5 with WebSearch, WebFetch, and local storage into a single agent.

💡 Why the SDK over a raw chat loop

It's for professionals who need stability, reproducibility, and low glue-code overhead — automating dev tasks, scripting analytics, or packaging a workflow into a reusable CLI tool.

Prerequisites

  • Python 3.10+
  • Node.js 18+
  • A Claude API key or Anthropic account

Install

1Install the Claude Code CLI

npm (macOS / Linux / Windows) 
npm i -g @anthropic-ai/claude-code
claude   # sign in
Windows (PowerShell alt) 
irm https://claude.ai/install.ps1 | iex
# add C:\Users\<you>\.local\bin to PATH, restart PowerShell, then:
claude --version

2Install the Agent SDK

bash 
pip install claude-agent-sdk

The Agentic Loop

Every agent the SDK builds runs the same feedback cycle. Keep it in mind as you wire up TrendSmith:

PhaseWhat happens
Gather contextRead the request and pull in what's needed (search results, fetched pages).
Take actionCall tools — WebSearch, WebFetch, write a file.
Verify workCheck the result against the goal before finishing.
RepeatLoop until the task is done.

Building TrendSmith

3Imports & configuration

Load the SDK types — AssistantMessage, ClaudeSDKClient, ToolUseBlock, ResultMessage — define a system prompt that guides behavior, and set the allowed tools to WebFetch and WebSearch.

4State management

Track per-request data: streamed text, model name, token usage, the tools invoked, and execution time. Reset it cleanly between queries so each command starts fresh.

5File handling

Save reports safely, falling back to the home directory if permissions fail. Sanitize filenames and auto-timestamp the saved markdown.

6The main event loop

Parse the user's command, stream the response asynchronously, auto-save trend reports to reports/<topic>_<timestamp>.md, and print a usage summary (model, tokens, tools used, duration, cost).

CommandWhat it does
/trend "<topic>"Multi-source scan (≤1 WebSearch + ≤2 WebFetch), auto-saved.
/scan "<query>"Quick one-source scan (≤1 WebFetch only).
/helpShow the commands.
/exitQuit.

Run It

bash 
python trend_smith.py

Try /trend "AI chip startups". The tutorial's run produced a full markdown report — signals, key players, risks, sourced information — at a cost of about $0.136.

⚠️ Watch out

Cap your tool calls (the /trend limits above exist for a reason) — an unbounded agentic loop fans out into many search/fetch calls and the per-run cost climbs fast. Set explicit limits and watch the usage summary.

Once the loop is yours, TrendSmith is just a template. Swap the tools and system prompt and the same gather–act–verify skeleton becomes any agent you need — a code reviewer, a research assistant, an automation runner.
